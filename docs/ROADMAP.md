# ClauseFlow Product Roadmap

ClauseFlow will remain one focused agreement protocol. New work must strengthen trustworthy settlement or real adoption instead of adding unrelated AI features.

## Submission Gate

- Verify a separate refund transaction on Bradbury and show it in public Dashboard totals.
- Perform the final clean deployment only after product approval, then repeat the paid and refunded smoke paths with real evidence.
- Keep the production frontend at [clauseflow-two.vercel.app](https://clauseflow-two.vercel.app) and verify wallet flows after the final clean deployment.
- Record a short end-to-end demo and publish a plain-language project post.

## Pilot Release

- Recruit a small group of Builders and Clients with public web deliverables.
- Add reusable agreement templates only from completed pilot patterns, with every clause editable before funding.
- Improve evidence findings with source-by-source accessibility, timestamps, criteria mapping, and concise missing-item explanations.
- Track pilot feedback through public issues and document protocol changes as milestones.

## Protocol Growth

- Add dispute and revision evidence without weakening deterministic deadline and settlement rules.
- Add an optional event indexer for faster discovery while preserving contract views as the source of truth.
- Build address-level work history and reputation from completed agreements, never from unverified profile claims.
- Support additional evidence types only when validators can retrieve authoritative public sources.

Success means recurring agreements between real parties, understandable review evidence, and settlement records that remain useful after the submission review ends.
